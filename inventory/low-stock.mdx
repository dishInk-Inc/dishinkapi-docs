Low Stock Alerts fire when any ingredient drops below the threshold you configured. Managers can act before an item runs out mid-service.

## Overview

- Threshold set per ingredient during creation.
- Alerts appear on the dashboard, POS home screen, and by email/SMS if enabled.
- Optional: block the sale of any product that requires an out-of-stock ingredient.

<Note>
  Alerts can be configured per branch. A central warehouse and one branch can have different thresholds for the same ingredient.
</Note>

## Tips & best practices

<Tip>
  Set thresholds at your **lead time** — e.g. if reorder takes 2 days and you use 3 kg/day, set the threshold at 6 kg minimum.
</Tip>

## Common mistakes

<Warning>
  Do not set thresholds too low to "reduce noise". You'll run out of stock and disappoint guests before the alert fires.
</Warning>
