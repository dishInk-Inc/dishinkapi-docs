Menu view modes control how your menu looks to customers when they visit your public menu page. You can choose which views are available and which one loads by default.

Go to **Front Operations → Settings → Menu View Modes** to configure this.

<Note>
  Menu view mode control is available on **Growth, Growth Pro, and Enterprise** plans. Starter plan accounts use the default view and cannot change it. [View upgrade options →](https://dishink.com/pricing)
</Note>

## The three view modes

| Mode | Description |
| --- | --- |
| **Category** | Groups items by category with section headers. Best for menus with many sections. |
| **Grid** | Displays items as cards in a photo grid. Best for visual, image-heavy menus. |
| **List** | Compact single-column list. Best for quick scanning and simple menus. |

<Note>
  View modes apply to the **Nightclub** and **Bold** templates. Other templates use their own fixed layout and are not affected by this setting.
</Note>

## How it affects the item detail page

The default view you choose also changes the layout of individual item detail pages:

- **List default** — the detail page shows no image, with a split layout: description on the left, actions on the right. Related items appear as a compact list.
- **Grid or Category default** — the detail page shows a full image hero and card layout.

## Quick Apply presets

The quickest way to configure view modes is to use one of the **Quick Apply** presets. Each preset sets both the enabled views and the default in one click.

Click any preset card to apply it instantly. The active preset is highlighted with a red border and a tick.

Common presets include combinations like Category only, Grid \+ Category, All three views, and List only — each with a short description of what it's best suited for.

## Customising manually

If no preset fits, you can configure enabled views and the default view independently.

### Enabling and disabling views

Under **Enabled Views**, each view mode has a checkbox. Uncheck a view to remove it from the switcher on your menu page. Customers will only see the views you have enabled.

<Warning>
  At least one view must always be enabled. If only one view is enabled, the switcher is hidden from customers entirely — they just see that view with no option to change it.
</Warning>

### Setting the default view

Under **Default View**, click the view you want customers to see first when they open your menu. Only enabled views appear here as options.

The default view is highlighted with a filled red background. Disabled views cannot be set as default.

## Reading the summary

At the bottom of the page a **Customer sees** summary shows exactly which views are active and which is the default (marked with ★). Use this to confirm your configuration before saving.

If only one view is enabled, the summary also notes that the switcher will be hidden.

## Tips & best practices

<Tip>
  If your menu is very image-heavy, set Grid as the default. Customers respond better to photos than text lists when browsing food.
</Tip>

<Tip>
  If you run a bar or quick-service counter with a short menu, List default gives customers the fastest way to scan and order. It also makes the item detail page load faster on slow connections.
</Tip>

<Tip>
  Use the Quick Apply presets first. Most restaurants fit one of the preset combinations — manual configuration is only needed for edge cases.
</Tip>

## Common mistakes

<Warning>
  Setting a view as default does not automatically enable it if it was previously disabled. Enable the view first under Enabled Views, then set it as default.
</Warning>

<Warning>
  This setting only affects the Nightclub and Bold templates. If you change the default view and nothing seems to change on your menu page, check which template is active in your website settings.
</Warning>
