The DishInk Waiter App puts your entire front-of-house operation in your team's pocket. Servers use it on any phone, tablet, POS terminal, or browser to build table carts, add modifiers, send orders directly to the kitchen, view live running bills, and request split payments — all without walking to a fixed POS terminal. No dedicated hardware required; your team can use their own personal devices or any device already in the venue.

## Full Service Workflow

<Steps>
  <Step title="Guest Seated">
    The waiter marks the table as occupied and is assigned to it. The table card turns active on the floor plan.
  </Step>
  <Step title="QR Browsed">
    The guest scans the table QR code to browse the digital menu with photos, prices, and descriptions on their own phone.
  </Step>
  <Step title="Cart Built">
    The waiter adds items to the table cart, selecting modifiers and attaching per-item notes. The cart saves locally so it survives any page refresh or device sleep.
  </Step>
  <Step title="Order Sent">
    When the table is ready, the waiter sends the full cart to the kitchen in one tap. All items, modifiers, notes, the table number, and the waiter's name fire to the KDS instantly.
  </Step>
  <Step title="Kitchen Notified">
    The KDS ticket appears in the kitchen with prep timers and full item detail. The customer can also scan the QR code on their receipt to track order progress in real time.
  </Step>
  <Step title="Food Served">
    The waiter marks the table status as Served on the floor plan. The status updates across every connected screen simultaneously.
  </Step>
  <Step title="Bill Requested">
    The waiter selects the split type and fires a pre-filled split request to the POS cashier, or requests a standard bill. The cashier sees it immediately and processes each payment.
  </Step>
  <Step title="Table Closed">
    Bill paid, table cleaned, and status returns to Available — ready for the next guest.
  </Step>
</Steps>

## Table Cart

Each table has its own persistent cart saved to local storage on the device. It survives page refreshes, network drops, and screen sleep — nothing is lost mid-service.

- Add or remove items at any point before sending
- Adjust quantities per item
- Add free-text notes to individual items
- Send the full cart in one tap when the table is ready
- Send per-person if guests prefer to confirm individually
- Clear and restart a cart at any point

A **red badge** on the table card on the floor plan shows when a table has unsent items stored locally, so nothing gets forgotten before a shift ends.

## Menu Modifiers

When a waiter adds any item to the cart, the modifier panel opens automatically. Every selection travels with the item to the KDS ticket — no verbal relay needed.

- **Modifier groups** — select from pre-configured groups such as crust type, extras, or cooking preference
- **Toggle selections** — tap to add or remove options across multiple groups
- **Per-item notes** — free-text field for special requests, allergen notes, or kitchen instructions
- **Price adjustments** — modifiers with cost differences update the item total automatically
- **Reusable groups** — create modifier groups once and attach them to multiple menu items

<Tip>
Modifier groups can be set as common (available on any item) or linked to specific menu items. Build your modifier library once in the Menu settings and reuse it across your full catalogue.
</Tip>

## Floor Plan & Table Filters

The Waiter App gives every server a live view of the entire floor — or just their own section.

- **My Tables** — shows only the tables assigned to the logged-in waiter
- **All Tables** — full floor view for managers and supervisors
- **Status filters** — filter by Ordering, Served, or Billing to prioritise what needs attention
- Every table card updates in real time with live data from the POS, KDS, and Table Management

Table statuses visible on the floor plan include: Available, Occupied, Ordering, Served, Billing, Reserved, Overdue (pulsing red), and Split Bill (purple border).

## Split Bill Request

When guests are ready to pay separately, the waiter handles the split request without leaving the floor.

1. Tap the split bill button on the table
2. Select the split type: equally, by item, or by custom amount
3. Fire the pre-filled request directly to the POS cashier
4. The cashier sees the split on their screen immediately and processes each payment

The waiter can also choose to let the cashier handle the split entirely — simply select the type the customers want and the POS cashier takes it from there.

## KDS Ticket Creation

Tapping **Send** fires the entire table cart to the kitchen as a structured ticket in under a second.

- All items sent simultaneously with correct quantities
- Every modifier and per-item note included on the ticket
- Table number and waiter name shown on the KDS screen
- Prep timers start the moment the ticket lands
- Supports order bumping and real-time status updates

No paper. No shouting. No re-keying at a separate terminal.

## Live Bill & Kitchen Status

The waiter can open any table at any time to see the running bill and the live kitchen status of every item — all without leaving the floor.

| Status | Meaning |
|---|---|
| **Queued** | Item sent to KDS, waiting for kitchen to start |
| **Cooking** | Kitchen has confirmed and started preparing |
| **Ready to Serve** | Item plated and waiting for collection |

The running bill total updates in real time as items are added or removed, giving the waiter full visibility before requesting payment.

<Note>
Cart data is saved to local storage so it persists on the device even without internet. Orders fire to the KDS using LAN mode as long as the device is on the same WiFi as the kitchen screen. Once connectivity is fully restored, everything syncs to the cloud automatically.
</Note>

## Frequently Asked Questions

<AccordionGroup>
  <Accordion title="Can I build separate orders per guest at a table?">
    Yes. Each person confirms their items and the waiter queues them individually in the cart. Send the full cart when everyone is ready, or send per-person if guests prefer to order at different times.
  </Accordion>
  <Accordion title="Can I edit cart items after adding them?">
    Yes — any time before the cart is sent. Add items, remove items, adjust quantities, and edit modifiers or notes freely. Once the cart is sent to the kitchen, new additions create a follow-up round on the same tab.
  </Accordion>
  <Accordion title="Does the Waiter App work offline?">
    Yes. The cart saves to local storage on the device, so it persists through any network drop. Orders fire to the KDS over LAN while on the same WiFi, even without internet. Everything syncs to the cloud when connectivity is restored.
  </Accordion>
  <Accordion title="Can waiters see only their own tables?">
    Yes. The floor view has a **My Tables** toggle that filters the floor plan to show only the tables assigned to the logged-in waiter. Switch to **All Tables** for the full floor view.
  </Accordion>
  <Accordion title="Do modifiers appear on the KDS ticket?">
    Yes. Every modifier, note, and special request appears in full on the KDS ticket. The kitchen has complete context for every item without any verbal relay from the floor.
  </Accordion>
</AccordionGroup>
