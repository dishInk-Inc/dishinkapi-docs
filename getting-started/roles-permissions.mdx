DishInk ships with predefined roles that match standard restaurant hierarchy. Enterprise administrators can also build custom permission sets in **Staff Management → Permissions**.

## Overview

| Role | Typical user | Access |
| --- | --- | --- |
| **Owner** | Business owner | Full access, billing, all branches |
| **Administrator** | IT / operations lead | Full access minus billing |
| **General Manager** | Branch GM | One or more branches, full ops |
| **Manager** | Shift manager | Overrides, refunds, voids, reports |
| **Cashier** | Front counter | POS, receipts, cash-out |
| **Waiter** | Floor staff | Waiter App, table service |
| **Kitchen** | Chef / line cook | KDS only |
| **Custom** | Any | Fine-grained permission set |

## Tips & best practices

<Tip>
  Follow the principle of least privilege. Only owners and administrators should have access to billing, integrations, and security.
</Tip>

## Common mistakes

<Warning>
  Do not give cashiers direct refund permissions. Require a manager PIN — the single most effective control against internal loss.
</Warning>
