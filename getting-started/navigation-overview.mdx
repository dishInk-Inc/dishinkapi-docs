DishInk shares a consistent layout across every module. Once you learn the pattern on one screen, the rest follow the same rules.

## Dashboard layout

<Frame caption="Dashboard navigation">
  ![Dashboard navigation](/images/getting-started/dashboard-nav.png)
</Frame>

| Area | Purpose |
| --- | --- |
| **Top bar** | Branch switcher, search, user menu, notifications |
| **Left sidebar** | Modules — POS, KDS, Waiter, Tables, Inventory, Reports |
| **Main pane** | The current module view |
| **Right rail** | Contextual actions, filters, live status |

## POS layout

- **Left** — categories and menu grid
- **Centre** — current order (cart)
- **Right** — payment panel and split billing controls
- **Bottom** — quick actions (discount, void, hold)

## Waiter App layout

- **Home** — assigned tables and floor plan
- **Table view** — cart, modifiers, send-to-kitchen
- **Notifications** — bill requests, kitchen status changes

## KDS layout

- **Hall View** — all active tickets
- **Station View** — filtered by kitchen station
- **Bump bar** — mark preparing, ready, or completed

<Note>
  All modules share a common colour language: green (fresh), amber (attention), red (overdue).
</Note>

## Tips & best practices

<Tip>
  Use the global search at the top of the dashboard to jump to any customer, order, product, or staff member — much faster than clicking through menus.
</Tip>

## Common mistakes

<Warning>
  On smaller tablets the sidebar collapses into a menu button. Expand it before assuming a feature is missing.
</Warning>
