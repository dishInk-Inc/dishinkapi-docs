## Overview

Generating payroll turns clocked hours, wages, tips, and deductions into pay slip records ready for review. DishInk calls the `payroll-operations` edge function with one of two actions:

- **`bulk_generate_payroll`** — create draft records for every active staff member in one go
- **`generate_payroll_record`** — create a single draft record for one staff member

Every generated record starts in **draft** state. You review, adjust, approve, then pay.

## Bulk generation

Use this for a normal pay period (weekly or monthly).

<Steps>
  <Step title="Open the Records tab">
    Go to **Payroll → Records**.
  </Step>
  <Step title="Click Generate">
    Opens the **Payroll Generation** modal.
  </Step>
  <Step title="Select Bulk">
    Choose the **All active staff** option (default).
  </Step>
  <Step title="Pick the period">
    Set the start and end dates. Default is the previous 7 days.
  </Step>
  <Step title="Confirm">
    Click **Generate**. DishInk creates one draft record per active staff member with wages, tips, and deductions calculated.
  </Step>
  <Step title="Review each record">
    Sort by net pay or hours worked and spot-check unusual values before approving.
  </Step>
</Steps>

## Single-staff generation

Use this to fix a missed staff member, run an off-cycle bonus, or regenerate one record after a wage change.

<Steps>
  <Step title="Open the Records tab">
    Go to **Payroll → Records**.
  </Step>
  <Step title="Click Generate → Single Staff">
    Choose the **Single staff member** option.
  </Step>
  <Step title="Pick the staff member">
    Search or select from the dropdown.
  </Step>
  <Step title="Pick the period">
    Set the start and end dates.
  </Step>
  <Step title="Generate">
    Creates a single draft record.
  </Step>
</Steps>

## What gets calculated

<CardGroup cols={2}>
  <Card title="Base pay" icon="clock">
    Hours × base rate (hourly), daily × days worked, or fixed monthly (with optional proration).
  </Card>

  <Card title="Overtime" icon="stopwatch">
    Hours above the OT threshold per shift × OT multiplier or absolute OT rate.
  </Card>

  <Card title="Tips" icon="hand-holding-dollar">
    Distributed pool share (`equal_split`) or direct-to-waiter total (`to_waiter`) for the period.
  </Card>

  <Card title="Deductions" icon="circle-minus">
    Active deductions applied at generation time (PAYE, NAPSA, custom).
  </Card>

  <Card title="Salary advances" icon="hand-holding">
    Outstanding balance for approved advances deducted from gross.
  </Card>

  <Card title="Commission" icon="percent">
    For commission wage types, commission % × attributed sales.
  </Card>
</CardGroup>

## Regenerating a record

Draft records can be regenerated freely — the old draft is replaced with fresh calculations. **Approved** or **paid** records cannot be regenerated; you must revert to draft first (see [Payroll Records](/payroll/records)).

<Note>
  Regenerating pulls the **current** wage structure, deductions, and tips. If you're mid-period, wait until the period ends and all shifts are closed before regenerating.
</Note>

## Tips

- Always **close every open shift** before generating — open shifts don't count tips or hours accurately.
- Run bulk generation on a **fixed pay day** (e.g. every 5th of the month) so staff plan around it.
- Use single-staff generation for **corrections** and bonuses without re-running the whole team.
- Sort the draft list by **net pay** to catch anomalies (K0 records, unusually high totals).

## Common mistakes

- **Generating with open shifts.** Hours and tips will be understated. Close all shifts first.
- **Generating twice for the same period without cleaning up.** Delete the old drafts before regenerating, or the list will double up.
- **Regenerating after approval.** Not allowed. Revert to draft first if you need to change the calculation.
- **Forgetting to add a wage** to a new hire. They generate with K0 base pay — always set the wage before their first pay period.
